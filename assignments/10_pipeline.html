The Graphics Pipeline

Not only have we build a fully-functioning basic 3d rendering engine, but this rendering engine is also a model of the modern rendering pipeline.

The relevant stages are:
  1. Vertex Specification: vertex data of 3d models is stored in arrays to be sent to later stages.
  2. Vertex Shader: model vertex coordinates are transformed to screen-relative coordinates based on transformations applied to the model, camera position, and additional effects.
  3. Face Culling: some triangles are discarded depending on which direction they're facing and which directions they should be drawn from (some triangles, for instance, can specify that they should be drawn from both sides).
  4. The Perspective Divide: x and y coordinates are divided by the z coordinate (actually the w coordinate, but we'll talk about that later).
  5. The Viewport Transform: x and y coordinates are transformed from spacial coordinates to pixel coordinates for display on the screen.
  6. Rasterization: Each triangle is divided up into pixels.
  7. Fragment Shader: Color for each pixel of each triangle is determined using an arbitrary formula.
  8. Depth Test: Each pixel is tested against the depth buffer to determine if it should be drawn.
